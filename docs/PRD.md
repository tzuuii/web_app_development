## 1. 專案概述

### 1.1 背景與動機

現代學生面臨資訊爆炸的挑戰：課堂講義越來越多、考試範圍越來越廣，卻缺乏有效的工具幫助整理、複習與自我評測。傳統的學習方式依賴手抄筆記與題庫刷題，效率低且缺乏個人化回饋。

本系統希望透過 AI 技術，**自動化整理學習內容、智慧出題與分析學習弱點**，讓學生可以把時間花在真正需要加強的地方，而非重複低效的瀏覽。

### 1.2 目標用戶

| 用戶類型 | 特徵描述 |
|---|---|
| 高中生 | 準備學測、指考，需要快速整理各科重點 |
| 大學生 | 期末考衝刺，需要消化大量講義與論文 |
| 考試備考者 | 準備公職、證照、研究所等考試 |
| 自學者 | 無老師指導，需要 AI 輔助建立學習計畫 |

### 1.3 核心價值主張

- ⚡ **省時**：上傳講義即可自動獲得摘要，不再手抄
- 🎯 **精準**：AI 分析答題記錄，精準定位弱點
- 📈 **持續進步**：每日學習追蹤，可視化進度讓成就感提升
- 🗣️ **自然互動**：語音問答如同有個 AI 家教隨時在旁
- 📚 **多科整合**：一個平台管理所有科目

## 2. 功能需求

### 功能一：自動整理筆記

**功能描述**：使用者上傳 PDF、圖片或文字格式的講義，系統透過 AI 自動生成結構化摘要、重點條列與關鍵字提取。

**使用者故事**：
> 作為一名大學生，我希望上傳課堂 PDF 後能自動獲得重點摘要，以便我不需要花兩小時手動整理筆記。

**驗收條件**：
- [ ] 支援上傳格式：PDF、JPG/PNG（OCR）、TXT
- [ ] AI 生成摘要段落、重點條列（bullet points）、關鍵字標籤
- [ ] 使用者可手動編輯與儲存 AI 產出的筆記
- [ ] 支援多份筆記依科目分類管理

---

### 功能二：AI 出題與測驗

**功能描述**：系統根據筆記內容自動生成選擇題與簡答題，並在使用者作答後即時批改與解析。

**使用者故事**：
> 作為一名高中生，我希望系統能根據我上傳的筆記自動出題，以便我可以測試自己是否真的理解內容。

**驗收條件**：
- [ ] 支援自動生成選擇題（4 選 1）與簡答題
- [ ] 作答完成後顯示答案解析與正確率
- [ ] 使用者可選擇題目數量與難度
- [ ] 測驗記錄自動儲存至歷史紀錄

---

### 功能三：學習進度追蹤

**功能描述**：系統記錄使用者每日的學習行為（閱讀時間、測驗次數、正確率），並以圖表呈現進度趨勢。

**使用者故事**：
> 作為一名備考者，我希望看到每週的學習時數與測驗成績趨勢圖，以便我調整學習策略並保持動力。

**驗收條件**：
- [ ] 記錄每日學習時間（進入系統 → 離開）
- [ ] 顯示每日/週/月正確率折線圖
- [ ] 顯示各科目學習時間比例圓餅圖
- [ ] 支援學習目標設定（例如每日 1 小時）

---

### 功能四：弱點分析與複習建議

**功能描述**：AI 分析使用者的歷次答題記錄，找出錯誤頻率高的知識點，主動推薦相關筆記與補充題目。

**使用者故事**：
> 作為一名自學者，我希望系統能告訴我哪些主題我的答對率最低，以便我優先複習那些弱項而非浪費時間在已掌握的內容。

**驗收條件**：
- [ ] 每次測驗結束後更新弱點統計
- [ ] 弱點分析頁面顯示各知識點的掌握程度（強/中/弱）
- [ ] 自動推薦對應弱點的複習筆記
- [ ] 每週生成「個人化複習計畫」

---

### 功能五：語音問答學習

**功能描述**：使用者可透過語音或文字向 AI 提問，AI 以對話式方式回答學科問題，如同互動式家教。

**使用者故事**：
> 作為一名高中生，我希望可以用說話的方式問 AI「牛頓第三定律是什麼意思」，以便我在不方便打字時也能學習。

**驗收條件**：
- [ ] 支援文字輸入問答（聊天介面）
- [ ] 支援語音輸入（Speech-to-Text）
- [ ] AI 回答可包含公式、圖表說明
- [ ] 對話紀錄可儲存與搜尋

---

### 功能六：多科目整合管理

**功能描述**：使用者可建立多個科目資料夾，所有筆記、測驗、進度記錄均依科目分類，並可跨科目搜尋。

**使用者故事**：
> 作為一名大學生，我希望在同一個平台同時管理數學、英文、程式三個科目的筆記，以便我不需要切換不同工具。

**驗收條件**：
- [ ] 支援建立自訂科目（含科目名稱、顏色標籤、圖示）
- [ ] 科目首頁顯示該科目的筆記數量、測驗次數、最近活動
- [ ] 全局搜尋功能跨科目搜尋筆記關鍵字

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|---|---|
| 後端框架 | Flask（Python） |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| AI 整合 | OpenAI API / Gemini API |
| 語音功能 | Web Speech API（前端）+ Whisper（後端 STT） |
| 前端 | HTML + Vanilla CSS + JavaScript |
| 檔案處理 | PyPDF2 / pdfplumber（PDF 解析）、pytesseract（OCR） |

### 3.2 效能考量

- **筆記摘要生成**：應在 30 秒內完成（一般 A4 講義 5 頁以內）
- **出題速度**：10 題應在 10 秒內完成
- **頁面載入**：首屏載入 < 3 秒（本地 SQLite 查詢）
- **並發支援**：支援 10 位以下同時使用（單機部署）

### 3.3 安全考量

- 使用者資料（筆記、測驗記錄）存於本地 SQLite，不上傳至第三方
- AI 請求僅傳送文字內容（不含個人識別資訊）
- 使用者帳號密碼以 bcrypt 加密儲存
- 上傳檔案類型白名單驗證（防止惡意檔案注入）

### 3.4 可用性考量

- 支援繁體中文介面
- 介面需在 1024px 以上螢幕正常顯示
- 所有主要功能操作步驟不超過 3 步

## 5. 系統頁面規劃（初步）

```
/                  → 首頁（登入前 Landing Page）
/register          → 註冊頁
/login             → 登入頁
/dashboard         → 學習儀表板（進度總覽）
/subjects          → 科目列表
/subjects/<id>     → 科目詳情（筆記列表 + 測驗記錄）
/notes/upload      → 上傳筆記
/notes/<id>        → 筆記詳情（AI 摘要 + 原文）
/quiz/generate     → 生成測驗
/quiz/<id>         → 進行測驗
/quiz/<id>/result  → 測驗結果與解析
/analysis          → 弱點分析與複習建議
/chat              → 語音/文字問答頁
```

## 7. 未解決問題與風險

| 風險項目 | 說明 | 緩解策略 |
|---|---|---|
| AI API 費用 | OpenAI 費用可能超出預算 | 優先測試免費 Gemini API，設置每日用量上限 |
| PDF 解析準確度 | 複雜排版的 PDF 可能解析錯誤 | 使用 pdfplumber，提供使用者手動修正介面 |
| 語音 STT 準確度 | 口音或專業術語可能辨識錯誤 | 允許使用者修正轉錄文字後再送出 |
| 開發時程 | 功能多、時間有限 | 嚴格依 MVP 優先級執行，Nice-to-Have 延後 |
